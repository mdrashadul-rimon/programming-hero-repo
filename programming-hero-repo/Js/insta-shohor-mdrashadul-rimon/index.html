Frequently Asked Question

Javascript is a scripting language. It is a client side programming language. Javascript works in both HTML file inside a <script> tag or we can use it by external .js file using src from a source. Javascript runs in browser by their own V8 engine which execute javascript code line by line. Every javascript engine has parser which parse javascript code line by line as I said and it is called interpreter. When parser get all code correct it send to AST (Abstract Syntax Tree). Then AST translet to machine code. When all the procedure and convertion happens then the javascript works.

Javascript has events like synchronous and asynchronous. Event loop is related to asynchronous events. Event loop is like run the process in the background wheather another call happens or not. Let say a process encounter a 5 seconds delay execution. But javascript execute line by line. So, other code will run in background as it is and after 5 seconds the delay code will execute. So this is how javascript event loop works.

localStorage localStorage is browsers local side only. It can not be accessible by server until GET or POST sent. localStorage has no expiration of data if user use or not. when user delete by javascript then it removed. localStorage is not safe. It is easily accessible by attacker with this single code localStorage.getItem(secret).
sessionStorage sessionStorage is a browser session based method. It store data when a session started and clear all the data when a session ends. Data stored in sessionStorage is not shared. sessionStorage can also be in attack. An attacker can easily access with a single line of code sessionStorage.getItem(secret).
